# Memory Vault Policy

Memory vault stores reusable engineering knowledge, not temporary logs.

## Store

- architecture decisions
- module boundaries
- recurring implementation patterns
- testing patterns
- release lessons
- post-mortem root causes
- integration conventions

## Do not store

- raw logs without summary
- secrets
- temporary TODOs
- duplicate Git history
- personal notes unrelated to the project

## Required updates

Update memory vault when a PR changes:

- architecture
- shared patterns
- release process
- dependency policy
- debugging/post-mortem learning
- module ownership or boundaries
