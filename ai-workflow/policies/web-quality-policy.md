# Web Quality Policy

Apply this policy to UI, frontend, and public web changes.

## Baseline checks

- Accessibility: semantic markup, labels, keyboard flow, visible focus, contrast.
- Performance: unnecessary JavaScript, image sizing, lazy loading, cache behavior, render blocking assets.
- Core Web Vitals risk: LCP, INP, CLS.
- Responsive behavior: mobile, tablet, desktop.
- SEO for public pages: title, description, headings, canonical, indexability, structured data when needed.
- Best practices: no console errors, no deprecated browser APIs, secure links, safe external resources.

## Evidence

PRs with UI changes should include screenshots or a short screen recording when feasible, plus validation notes.
